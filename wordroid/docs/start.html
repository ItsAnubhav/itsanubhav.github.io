Getting Started

Installing JDK

1. Click here to go to download page

2. Click on Accept License Agreement checkbox



3. Now choose the version according to your operating system and download will start.

4. After downloading, install it as you install other softwares.

Installing Android Studio

This project is compatible with Android Studio. So if you don't already have then go ahead and download Android Studio

1. Click here to go to download page

2. Click on green Download Android Studio button.



3. Now a dialog appears which will ask you to accept terms and condition. Check the checkbox saying I have read and agree with the terms and conditions and then click on Download Android Studio

4. After the download complete, install it as you usually do.

Import Project

1. Extract the template file.

2. Open Android Studio.

3. Click on Open an existing Android Studio Project



4. Now locate the extracted template file and click on OK



5. After that, Android Stuido will sync the project and will download all the dependencies. Make sure you are connected to internet. This will take a few minutes or so. Depending on your internet speed.

Order of SignIn/SgnUp Providers

Ther provders will be dsplayed in the same order they are in code. So if you want to put Facebook Login button at the top of the screen then just move it to the top of the list in code.

For Facebook or Twitter Authentication Provider

Facebook and Twitter authetication are disabled by default because they insreases the APK size. If you want to enable them then just go to build.gradle file and check line number 75

//Facebook SDK
//implementation 'com.facebook.android:facebook-login:[4,5)'
//Twitter SDK
//compile 'com.twitter.sdk.android:twitter-core:3.1.1'
Just uncomment the line below Facebook SDK if you want to enable facebook login and Twitter SDK for twitter login
App Updates
    public static final int currentAppVersion = 4;
    public static final boolean isPluginInstalled = true;
The currentAppVersion represents the current version of your app. This is helpful to check if a new update is live or not. When you update the value of app version in your wordpress dashboard, all the installed app compares their currentAppVersion with the value you updated. And if the installed app's currentAppVersion is lower than the server returned value then users will see a message to update the app.
You can specify which message to show. Either to force update or not. Or if you dont want to use this feature then set both the value to same number and never update them.
